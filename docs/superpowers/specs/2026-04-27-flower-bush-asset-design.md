# 花丛资产设计

## 背景

第 11-14 天计划新增花丛表现，但当前仓库中没有可直接接入的低模花丛模型。

现有项目已经有两类可复用模式：

- 真实模型加载并归一化：`LeafyTree.ts`
- 程序化占位/替代几何体：`LowPolyWideTree.ts` 中的占位实现思路

为了尽快落地第 11-14 天内容，并保持性能可控，本次先实现一个程序化低模花丛资产文件。

## 目标

- 新增 `FlowerBush.ts`，提供一个可直接实例化的低模花丛资产。
- 第一版不依赖外部 glTF 资源，完全使用 Three.js 基础几何体拼装。
- 视觉上区分于现有灌木：更矮、更紧凑、带明显花朵颜色点缀。
- 保留后续替换为真实模型的接口形状，避免未来重写接入逻辑。

## 范围

本次只实现花丛资产文件本身：

- `src/utils/planet/assets/FlowerBush.ts`

本次不做：

- `VegetationLayer` 接入
- 第 11-14 天逐日节奏配置
- 外部花丛模型搜索与替换

## 方案

采用“方案 C”：

- 当前版本先实现程序化低模花丛
- 代码结构直接按“可替换真实模型”的方式组织

花丛内部结构：

- 底部叶团：`2-3` 个低段数球体，形成矮灌木基底
- 花朵主体：`3-5` 朵小花，围绕顶部和侧边分布
- 单朵花：使用小球花心 + 两组交叉花瓣片，形成低面数花感

## 接口设计

导出以下接口：

- `preloadFlowerBushTemplate(): Promise<Group>`
- `createFlowerBushInstance(options?: { targetHeight?: number; rotationY?: number; paletteVariant?: 'pink' | 'yellow' | 'blue' }): Group`

说明：

- `preloadFlowerBushTemplate()` 第一版可以直接返回程序化模板，但保留异步接口，便于未来切到 glTF 加载
- `paletteVariant` 用于快速切换花朵配色，避免花丛实例看起来完全重复

## 视觉要求

- 高度建议基准为 `0.35-0.5`
- 叶团主色沿用当前星球绿色系，避免与现有树和灌木冲突
- 花朵颜色第一版提供三套：粉色、黄色、蓝紫色
- 花丛整体视觉应比当前灌木更轻巧，不抢树的主体地位

## 性能要求

- 单个花丛只使用少量基础几何体
- 禁止引入贴图
- 禁止引入额外模型文件
- 优先复用少量材质实例，避免不必要的材质膨胀

## 测试要求

至少补一组资产级测试，覆盖：

- `createFlowerBushInstance()` 能正常返回 `Group`
- 传入不同 `targetHeight` 时实例缩放正确变化
- 传入不同 `paletteVariant` 时花朵颜色可区分

## 风险与约束

- 第一版是风格化替代资产，不追求真实花丛细节
- 若后续要替换为真实模型，应尽量保持导出接口不变
- 本次不接入运行时场景，因此只验证资产构造正确性
